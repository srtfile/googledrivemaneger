Google OAuth Setup
DataVault needs a Google Cloud OAuth 2.0 Client ID to access Google Drive. This one-time setup takes about 5 minutes.
1
Create a Google Cloud Project
Go to console.cloud.google.com, create a new project, then go to APIs & Services → Library and enable the Google Drive API.
2
Configure OAuth Consent Screen
Go to APIs & Services → OAuth consent screen. Set type to External, fill in the app name and your email, save. Under Scopes, add: https://www.googleapis.com/auth/drive.file
3
Create OAuth 2.0 Credentials
Go to Credentials → Create Credentials → OAuth 2.0 Client ID. Choose Web application. Under Authorized JavaScript origins, add the URL you're opening this file from (e.g. http://localhost or your HTTPS domain). Copy the Client ID.
4
Add Yourself as a Test User
Under OAuth consent screen → Test users, add your Google account email. This lets you sign in while the app is in "Testing" publishing status.
Your Google OAuth Client ID
The Client ID is stored in sessionStorage — only for this browser tab. You'll need to re-enter it if you close the tab (or hardcode it below).
