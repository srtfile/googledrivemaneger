Google OAuth Setup
DataVault needs a Google Cloud OAuth 2.0 Client ID to access Google Drive. Follow these steps once — it takes about 5 minutes.
1
Create a Google Cloud Project
Go to console.cloud.google.com, create a new project (or select an existing one), then navigate to APIs & Services → Library and enable the Google Drive API.
2
Configure OAuth Consent Screen
Go to APIs & Services → OAuth consent screen. Set type to External, fill in the app name (e.g. DataVault), your email, and save. Under Scopes add: https://www.googleapis.com/auth/drive.file
3
Create OAuth 2.0 Credentials
Go to APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID. Choose Web application. Under Authorized JavaScript origins add the URL where you'll open this HTML file (e.g. http://localhost or your site URL). Copy the Client ID.
4
Add Test Users (while in Testing mode)
Under OAuth consent screen → Test users, add your Google account email so you can sign in during testing.
Your Google OAuth Client ID
The Client ID is stored only in your browser's sessionStorage for this session.
